## 项目概述

这是一个外卖店铺运营指南项目，由呈尚营销策划有限公司创建，专注于美团、饿了么等外卖平台的精细化代运营服务。项目提供系统化的运营解决方案，帮助餐饮商家提升线上运营效率和销售转化率。

## 技术架构

### 前端技术栈
- **HTML5**: 使用语义化标签构建页面结构
- **CSS3**: 内联样式，使用CSS变量实现主题管理
- **原生JavaScript**: 无外部框架依赖，纯DOM操作
- **响应式设计**: 支持PC端和移动端自适应

### 外部依赖
- **Google Fonts**: Noto Sans SC 中文字体 (CDN)
- **Remix Icon**: 矢量图标库 (CDN)

## 开发指南

### 文件管理原则
- **交付HTML文件**: 根目录 `index.html` / `eleme-operation-guide.html` 为单文件页面，CSS 与 JavaScript 均内联，支持 GitHub Pages 部署与离线打开
- **源码维护方式**: `src/pages/*` 下按 `template + style + script + sections` 拆分维护，通过构建脚本生成交付 HTML
- **样式系统**: 使用CSS变量 (`:root`) 管理主题色彩和样式
- **响应式设计**: 移动优先的设计方法，断点为768px

### 设计系统规范
遵循 `.cursor/rules/design-system.mdc` 中定义的设计规范：

#### 色彩系统
- **主色**: `#3c64b1` (美团页面) / `#0066ff` (饿了么页面)
- **辅助色**: `#36b37e`
- **强调色**: `#ff9f43`
- **饿了么品牌色**: `#0066ff` (蓝), `#ff6600` (橙)

#### 字体规范
- **主字体**: 'Noto Sans SC', 'Helvetica Neue', Arial, sans-serif
- **标题**: 700字重，2.75rem (44px) 用于主标题
- **正文**: 400字重，1.6行高

#### 间距系统
- **基础单位**: 8px
- **内容间距**: 16px, 24px, 32px
- **圆角**: 12px (卡片), 统一的 `--border-radius` 变量

### 页面结构规范

#### 头部设计
- **渐变背景**: 135度线性渐变
- **装饰元素**: 使用伪元素创建圆形装饰
- **容器**: 最大宽度1200px，居中对齐

#### 内容卡片
- **背景**: 白色 `#ffffff`
- **阴影**: `0 10px 15px -3px rgba(0, 0, 0, 0.05), 0 4px 6px -2px rgba(0, 0, 0, 0.025)`
- **内边距**: 统一的卡片内边距
- **过渡效果**: `all 0.3s ease`

## 开发工作流

### 本地开发
```bash
# 1) 修改拆分源码（src/pages/*）
# 2) 一键生成单文件 HTML
python scripts/build_pages.py

# 3) 预览方式二选一
# 方式A：直接双击打开 index.html / eleme-operation-guide.html
# 方式B：使用本地服务器（可选）
python -m http.server 8000
# 访问 http://localhost:8000
```

### 文件编辑注意事项
- **HTML文件规模**: 由于内联CSS和JS，文件较大，编辑时注意性能
- **CSS变量**: 所有样式通过CSS变量管理，修改颜色等属性时优先更新 `:root` 变量
- **响应式测试**: 编辑后务必测试移动端和桌面端显示效果
- **浏览器兼容**: 确保在Chrome、Firefox、Safari、Edge等现代浏览器中正常显示

### 部署指南
- **静态托管**: 可直接部署到任何静态托管服务
- **CDN要求**: 需要网络连接访问Google Fonts和Remix Icon CDN
- **文件结构**: 保持当前文件结构，无需构建步骤

### 版本控制
- **Git历史**: 项目已有完整的提交历史
- **文件格式**: 支持HTML、Markdown、文本等多种格式
- **分支管理**: 主要开发在main分支进行

## 内容更新指南

### ⚠️ 内容格式规范（重要！）

**所有运营模块已经过精简优化，请严格遵循以下格式，不要随意添加冗余内容！**

#### 标准模块结构（三段式）

每个运营模块 `<section>` 必须采用以下精简格式：

```html
<section class="section" id="sectionX">
    <!-- 1. 标题 + 流程标签 -->
    <h2 class="section-title">
        <i class="ri-icon-name"></i>X. 模块名称
        <span style="background: linear-gradient(...);">第X天流程</span>
    </h2>

    <!-- 2. 软件操作指引（紫色渐变背景框） -->
    <div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); ...">
        <h3 style="color: white; ...">
            <i class="ri-computer-line"></i>软件操作指引
        </h3>
        <p style="margin: 0; ...">
            📍 请在软件中点击 <strong>运营工具</strong> → <strong>"具体功能名称"</strong> ...
        </p>
    </div>

    <!-- 3. 单段落精简描述（100-150字） -->
    <p style="color: var(--text-color); line-height: 1.8; font-size: 1rem; margin-top: 20px;">
        通过XX方法...实现XX目标...
    </p>
</section>
```

#### ❌ 严格禁止的格式

**绝对不要添加**以下内容：
- ❌ `<div class="subsection">` 子章节
- ❌ `<h3>目标</h3>` + `<ul>` 详细目标列表
- ❌ `<h3>措施</h3>` + `<ol>` 详细措施列表
- ❌ 任何多层级的详细说明
- ❌ 超过150字的长篇描述

#### ✅ 内容编辑原则

1. **精简至上**: 每个模块仅用一段话（100-150字）概括全部内容
2. **格式统一**: 所有22个模块必须保持完全一致的结构
3. **突出关键**: 只保留最核心的操作路径和目标
4. **避免冗余**: 详细内容已在 `餐饮店铺运营指南.mdc` 中保存，HTML仅展示精简版
